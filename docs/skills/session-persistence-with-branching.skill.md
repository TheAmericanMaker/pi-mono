# Session Persistence with Branching

## Purpose

Persist agent conversations to disk in a format that supports branching (exploring alternative conversation paths), forking (creating new sessions from any point), tree navigation (jumping between branches), and crash recovery. Uses append-only JSONL files for simplicity and durability.

## Dependencies

- [Message Transformation Pipeline](./message-transformation-pipeline.skill.md) — message types stored in sessions

## Core Concepts

### JSONL Format

Each session is stored as a JSONL file (one JSON object per line). The first line is a session header; subsequent lines are entries representing messages, model changes, compactions, and other events. Each entry is appended atomically (one write per line), making the format crash-resistant.

### Entry Linking (Parent IDs)

Each entry has a unique `id` and a `parentId` pointing to the previous entry. This creates a linked list within a session file. When combined with session forking, entries across multiple files form a tree.

### Session Tree

Multiple session files can be related through forking. Session B's header references Session A as its parent. The tree structure allows navigating to any point in any branch of the conversation history.

### Branching vs Forking

- **Branching** happens within a session tree when you navigate to an old entry and continue from there
- **Forking** creates a new session file that starts from a specific entry in an existing session

### Compaction Entries

When context is compacted (see Context Window Management), a compaction entry records the summary and which entries were kept. This allows reconstructing the current conversation state from the session file.

## Algorithm / Flow

### Creating a New Session

```
function newSession(cwd, options?):
    sessionId = options?.id or generateUUID()
    sessionDir = getSessionDir(cwd)
    ensureDirectoryExists(sessionDir)

    header = SessionHeader {
        type: "session",
        version: CURRENT_SESSION_VERSION,
        id: sessionId,
        timestamp: now().toISO8601(),
        cwd: cwd,
        parentSession: options?.parentSession
    }

    path = join(sessionDir, sessionId + ".jsonl")
    writeLineToFile(path, serialize(header))
    return sessionId
```

### Appending an Entry

```
function appendEntry(sessionPath, entry):
    // Assign ID if not set
    if not entry.id:
        entry.id = generateUUID()

    // Set timestamp
    if not entry.timestamp:
        entry.timestamp = now().toISO8601()

    // Atomic append (one line)
    line = serialize(entry) + "\n"
    appendToFile(sessionPath, line)
    return entry.id
```

### Loading a Session

```
function loadSession(sessionPath):
    lines = readFileLines(sessionPath)
    if lines is empty:
        error("Empty session file")

    header = deserialize(lines[0])
    if header.type != "session":
        error("Invalid session header")

    // Apply migrations if needed
    if header.version < CURRENT_SESSION_VERSION:
        header = migrateHeader(header)

    entries = []
    for i from 1 to len(lines) - 1:
        line = lines[i].trim()
        if line is empty: continue  // Skip blank lines
        entry = deserialize(line)
        entries.append(entry)

    return { header, entries }
```

### Building Current Context from Entries

```
function buildSessionContext(entries):
    // Find the last compaction entry (if any)
    lastCompactionIdx = -1
    for i from len(entries) - 1 down to 0:
        if entries[i].type == "compaction":
            lastCompactionIdx = i
            break

    messages = []

    if lastCompactionIdx >= 0:
        // Start with compaction summary as a user message
        compaction = entries[lastCompactionIdx]
        messages.append(createCompactionSummaryMessage(
            compaction.summary, compaction.tokensBefore, compaction.timestamp
        ))

        // Find the first kept entry and add messages from there
        startIdx = lastCompactionIdx + 1
        for i from lastCompactionIdx + 1 to len(entries) - 1:
            if entries[i].id == compaction.firstKeptEntryId:
                startIdx = i
                break

        for i from startIdx to len(entries) - 1:
            msg = getMessageFromEntry(entries[i])
            if msg: messages.append(msg)
    else:
        // No compaction — all entries contribute
        for entry in entries:
            msg = getMessageFromEntry(entry)
            if msg: messages.append(msg)

    return messages

function getMessageFromEntry(entry):
    switch entry.type:
        case "message": return entry.message
        case "custom_message": return createCustomMessage(entry)
        case "branch_summary": return createBranchSummaryMessage(entry)
        case "compaction": return createCompactionSummaryMessage(entry)
        case "thinking_level_change": return null  // Metadata only
        case "model_change": return null            // Metadata only
```

### Forking a Session

```
function forkSession(originalSessionPath, fromEntryId, cwd):
    // 1. Load original session
    original = loadSession(originalSessionPath)

    // 2. Create new session with parent reference
    newSessionId = newSession(cwd, {
        parentSession: original.header.id
    })

    // 3. Copy entries up to (and including) fromEntryId
    for entry in original.entries:
        appendEntry(newSessionPath, entry)
        if entry.id == fromEntryId:
            break

    return newSessionId
```

### Tree Navigation

```
function buildSessionTree(sessionDir):
    // 1. Load all session files in directory
    sessions = []
    for file in listFiles(sessionDir, "*.jsonl"):
        session = loadSession(file)
        sessions.append(session)

    // 2. Build parent-child relationships
    tree = new Tree()
    for session in sessions:
        node = tree.addNode(session.header.id, session)
        if session.header.parentSession:
            tree.addEdge(session.header.parentSession, session.header.id)

    // 3. Within each session, entries form a linked list via parentId
    for session in sessions:
        for entry in session.entries:
            tree.addEntryNode(session.header.id, entry)

    return tree

function navigateToEntry(tree, targetEntryId):
    // Find which session contains this entry
    session, entryIndex = tree.findEntry(targetEntryId)

    // Build messages from start of session up to this entry
    entries = session.entries[0:entryIndex + 1]
    messages = buildSessionContext(entries)

    return { sessionId: session.header.id, messages, entryIndex }
```

### Session Versioning & Migration

```
CURRENT_SESSION_VERSION = 3

function migrateHeader(header):
    if header.version is undefined:
        // v1: no version field
        header.version = 1

    if header.version == 1:
        // v1 → v2: add cwd field if missing
        if not header.cwd:
            header.cwd = "/"
        header.version = 2

    if header.version == 2:
        // v2 → v3: normalize timestamps
        header.timestamp = normalizeTimestamp(header.timestamp)
        header.version = 3

    return header
```

### Listing Sessions

```
function listSessions(cwd):
    sessionDir = getSessionDir(cwd)
    if not directoryExists(sessionDir):
        return []

    sessions = []
    for file in listFiles(sessionDir, "*.jsonl"):
        // Quick parse: read only first and last lines
        header = deserialize(readFirstLine(file))
        lastLine = readLastLine(file)
        lastEntry = deserialize(lastLine)

        sessions.append(SessionInfo {
            id: header.id,
            path: file,
            timestamp: header.timestamp,
            cwd: header.cwd,
            parentSession: header.parentSession,
            entryCount: countLines(file) - 1,  // Minus header
            lastEntryTimestamp: lastEntry.timestamp
        })

    // Sort by last entry timestamp, most recent first
    sessions.sort(by: lastEntryTimestamp, descending)
    return sessions
```

## Key Design Decisions

1. **Append-only JSONL** — Never modifies existing lines. This is crash-safe (partial writes only lose the last entry), simple to implement, and human-readable for debugging. Each line is a complete JSON object.

2. **Parent ID linking** — Rather than relying on line order (which could be corrupted), each entry explicitly links to its parent. This makes tree reconstruction robust.

3. **CWD-scoped directories** — Sessions are organized by working directory. This keeps project-specific sessions separate and makes cleanup easy (delete the directory to remove all sessions for a project).

4. **Separate files per fork** — Forking creates a new file rather than appending to the original. This avoids file locking between sessions and keeps each file self-contained (with copies of shared entries).

5. **Versioned schema** — The version field in headers enables forward-compatible evolution. Old sessions are migrated on load; new fields have defaults for backward compatibility.

6. **Metadata entries** — Model changes and thinking level changes are stored as entries (not as fields on messages) so they appear in the timeline and can be navigated to.

## Edge Cases & Failure Modes

| Scenario | Handling |
|---|---|
| Crash during append | Partial last line; skip malformed lines on load |
| Corrupt JSON line | Skip with warning; continue loading remaining entries |
| Missing parent session file | Load as standalone session (parentSession is informational) |
| Circular parentId references | Detect during tree building; break cycle |
| Very large session file | Stream-parse JSONL line by line rather than loading all into memory |
| Concurrent appends | Each append is a single write; filesystem provides atomicity for small writes |
| Session directory doesn't exist | Create on first session; return empty list when listing |
| Forking from compacted entry | Copy entries including compaction; new session starts with summary |
| Entry ID collision | UUIDs make this astronomically unlikely; detect and regenerate |
| Encoding issues | JSONL uses UTF-8; handle BOM if present |

## Testing Strategy

1. **Unit: Create and load** — Create session, append entries, load and verify all entries present
2. **Unit: Entry linking** — Verify parentId chain is correct for sequential appends
3. **Unit: Context building** — Build messages from entries; verify correct conversion
4. **Unit: Context with compaction** — Session with compaction entry; verify summary + kept messages
5. **Unit: Fork** — Fork from mid-session; verify new session has correct entries and parent reference
6. **Unit: Tree building** — Multiple sessions with forks; verify tree structure
7. **Unit: Navigation** — Navigate to specific entry; verify correct messages
8. **Unit: Migration** — Load v1 session; verify migrated to current version
9. **Unit: Corrupt lines** — Session with malformed JSON line; verify graceful skip
10. **Unit: Listing** — Multiple sessions; verify sorted by recency
11. **Integration: Crash recovery** — Simulate crash mid-append; verify session loads with last good entry
12. **Integration: Full workflow** — Create → append → compact → fork → navigate → resume
